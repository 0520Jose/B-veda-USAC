### 4. Diagnóstico Técnico y Conceptos Clave de Arquitectura Cloud
- **Definición de [[Cloud Computing]]:** Modelo que permite el acceso ubicuo, bajo demanda y a través de la red a un conjunto compartido de recursos computacionales configurables (servidores, almacenamiento, redes, aplicaciones y servicios) que pueden ser aprovisionados rápidamente con un esfuerzo mínimo de gestión o interacción por parte del proveedor de servicios.
- **Tipos de Nube según su Implementación:**
  - *Nube Pública:* Infraestructura abierta y accesible por el público o grandes grupos de la industria, operada por un proveedor externo de servicios cloud (como [[Amazon Web Services]], [[Microsoft Azure]] o [[Google Cloud Platform]]).
  - *Nube Privada:* Infraestructura operada exclusivamente para una única organización, ya sea administrada por la propia institución o por un tercero (en sitio o fuera de sitio).
  - *Nube Híbrida:* Composición de dos o más infraestructuras de nube (privada y pública) que permanecen como entidades únicas pero vinculadas por tecnología estandarizada para la portabilidad de datos y aplicaciones.
- **[[Serverless Computing]] y Cobro por Consumo:** Modelo de ejecución en la nube donde el proveedor de la plataforma administra dinámicamente el aprovisionamiento de los servidores físicos y virtuales. El desarrollador despliega bloques de código (funciones en [[AWS Lambda]]) que se ejecutan únicamente en respuesta a eventos específicos, facturándose exclusivamente en intervalos de milisegundos por el consumo de cómputo real y eliminando los costos de servidores inactivos.
- **Diferenciación Conceptual en Cómputo de AWS:**
  - **[[Amazon EC2]] (Elastic Compute Cloud):** Servicio de infraestructura como servicio ([[IaaS]]) que proporciona capacidad de cómputo escalable en la nube a través de instancias virtuales (servidores completos con sistema operativo a elección).
  - **[[Amazon ECS]] (Elastic Container Service):** Servicio altamente escalable de orquestación de contenedores que permite ejecutar y administrar aplicaciones contenerizadas con [[Docker]] dentro de un clúster de instancias o en modo *serverless* (AWS Fargate).
- **Hardening y Gestión de Identidades en AWS (`IAM`):**
  - El servicio [[IAM]] controla las autorizaciones y autenticaciones dentro de una cuenta de AWS.
  - *Regla Crítica de Seguridad:* Está absolutamente desaconsejado y constituye un riesgo grave utilizar las credenciales del **Usuario Raíz (*Root User*)** (correo electrónico y contraseña principales de la cuenta) para las operaciones cotidianas, acceso desde consola CLI o despliegue de software. El usuario raíz debe asegurarse con Autenticación Multifactor ([[MFA]]) y reservarse exclusivamente para tareas administrativas contables o de facturación críticas. Toda operación técnica debe ejecutarse mediante usuarios IAM o roles temporales con privilegios mínimos garantizados.
- **Contenedores vs. Máquinas Virtuales ([[Docker]]):**
  - Un contenedor Docker es un paquete ligero, ejecutable e independiente que incluye todo lo necesario para ejecutar una aplicación: código, tiempo de ejecución, herramientas del sistema, librerías y configuraciones. A diferencia de las máquinas virtuales tradicionales que virtualizan el hardware completo y un sistema operativo invitado pesado, los contenedores virtualizan a nivel del sistema operativo anfitrión, garantizando aislamiento, portabilidad y arranques casi instantáneos.

## Fórmulas, Sintaxis o Puntos Clave

```bash
# Comandos fundamentales para verificación e interactividad con Docker y AWS CLI

# 1. Comandos de verificación y empaquetado con Docker
docker --version
docker info

# Construcción de una imagen Docker a partir de un Dockerfile en el directorio actual
docker build -t mi_aplicacion_seminario:v1.0 .

# Ejecución de un contenedor aislado mapeando el puerto 8080 del host al 80 del contenedor
docker run -d --name app_laboratorio -p 8080:80 mi_aplicacion_seminario:v1.0

# Verificación de contenedores activos en ejecución y bitácoras del contenedor
docker ps
docker logs app_laboratorio

aws s3 ls

aws ec2 describe-instances --query "Reservations[*].Instances[*].[InstanceId,InstanceType,State.Name]" --output table
```
